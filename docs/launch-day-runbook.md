# Humi 1.0.0 首发发布 Runbook

更新日期：2026-06-30  
执行设备：codex@mbp-m5pro

## 1. 当前状态

- 小程序版本：`1.0.0`
- 版本描述：`首发上线候选：自建AI推荐、微信登录、三餐计划、清单库存闭环`
- AppID：`wx4040b89f3b363416`
- H5：`https://www.humi-home.com/`
- API：`https://api.humi-home.com`
- 当前状态：微信公众平台审核中

## 2. 审核通过后立即执行

1. 微信公众平台进入“小程序版本管理”。
2. 找到审核通过的 `1.0.0`。
3. 点击“发布”。
4. 发布后等待 3-10 分钟，再进行真机验证。
5. 在 AI-HQ Humi 状态中记录发布时间、执行人和结果。

## 3. 发布后真机验收

必须用真实微信扫码或搜索小程序验证，不只看开发者工具。

### P0 验收路径

- 打开小程序后可进入 Humi 首页。
- 首页能看到推荐菜单。
- 点击“今晚就做”后，菜单进入今日菜单。
- 点击“换一组”后，推荐能更新或本地兜底正常工作。
- 进入计划页，可以看到早餐、午餐、晚餐三餐槽。
- 进入清单页，可以看到食材清单。
- 勾选清单食材后，可进入家中库存。
- 进入“我的家”，能看到微信登录状态。
- 点击退出登录后，可以重新触发微信登录。

### 生产接口验收

```bash
npm run monitor:prod
```

期望：

- H5 返回 200。
- API `/health` 返回 `{"ok":true,"service":"humi-api"}`。
- API `/recommend` 返回 `source:"deepseek"`。

## 4. 发布后 24 小时监控

### 每 2-4 小时看一次

- `https://www.humi-home.com/` 是否可打开。
- `https://api.humi-home.com/health` 是否正常。
- 首页推荐是否有明显失败。
- 是否出现登录后一直“同步中”。
- 是否有用户反馈空白页、打不开、无法提交菜单。

### 需要立即处理的 P0

- 小程序无法打开 H5。
- 微信登录完全失败，且游客模式无法继续。
- 推荐接口持续失败且本地兜底不可用。
- 菜单/清单核心链路无法完成。
- 隐私、登录、手机号授权相关审核或用户投诉问题。

## 5. 不建议在审核通过前/刚发布后做的事

- 不要上传新版本覆盖 `1.0.0`，除非审核反馈要求。
- 不要清退 Supabase 代码。
- 不要改登录、权限、手机号、生产 API 架构。
- 不要大改 UI 或菜谱数据。
- 不要引入新的审核敏感能力，比如支付、订阅消息、社区、UGC。

## 6. 1.0.1 触发条件

出现以下情况才上传 1.0.1：

- 审核反馈必须修改。
- P0 核心链路阻塞。
- 明显合规/隐私问题。
- 多个首批用户遇到同一处不可继续的问题。

普通文案、视觉细节、推荐口味不准，先记入 `docs/launch-feedback-and-101-backlog.md`，不急着发版。
