# Humi 数据与隐私清单 V1

更新日期：2026-06-29

## 数据最小化原则

- 游客优先体验，不强制注册。
- 只处理完成菜单、清单、库存和推荐所需的数据。
- 过敏和忌口只用于避开菜品，不作医疗推断。
- 推荐服务不接收用户或家庭成员邮箱。
- 产品埋点使用随机会话标识，不读取微信身份。
- 暂不申请位置、相册、摄像头、麦克风和通讯录权限。
- 小程序首发支持微信登录；手机号绑定作为可选账号安全能力，仅在用户主动点击“绑定手机号”并授权后触发。

## 第三方服务

### 云端数据库与认证

用于账号认证、家庭数据同步、事件记录和服务端函数。

### 晚饭推荐服务

通过服务端函数调用，只发送完成推荐所需的结构化上下文。API 密钥不进入前端。

### 静态站点与正式 H5 托管

用于加载 Humi 页面、图标和菜品图片。正式上线后以备案域名对应的托管服务为准。

## 保留与删除

- 游客本地数据：由用户设备/WebView 保存；清理缓存后可能删除。
- 登录数据：在账号和家庭空间存续期间保存。
- 当前产品尚未提供自助注销和一键删除入口。
- 已提供有效联系渠道：`humihome2026@163.com`。云端账号数据删除申请先通过人工邮箱流程处理。
- 后续上线账号注销时，应同步更新产品、协议和审核材料。
- 手机号绑定后，前端只展示脱敏手机号。账号数据删除申请可通过联系邮箱人工处理，处理时同步解除微信身份和手机号绑定。
