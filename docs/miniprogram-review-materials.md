## 1. 基本信息

### 小程序名称

Humi

### 简称

Humi

### 一句话介绍

帮家里安排晚饭、整理食材清单和厨房库存。

### 详细介绍

Humi 是一款家庭饮食安排工具。用户可以先体验“今晚吃什么”推荐，将合适的菜加入今晚菜单或一周计划，并根据菜单生成食材清单。用户也可以记录厨房库存、家庭口味、忌口和饮食目标，让后续安排更贴近家庭实际情况。

Humi 提供的是生活安排和信息整理能力，不提供医疗诊断、治疗建议或专业营养咨询。

## 2. 服务类目候选

最终类目必须以微信公众平台当前可选项和主体资质为准，不在提交前凭文档猜定。

建议按以下顺序检查：

1. 工具类中与生活、效率或信息管理相关的类目。
2. 生活服务中与家政、家庭生活相关的类目。
3. 餐饮相关类目仅在后台要求且主体资质满足时选择。

避免把 Humi 描述为：

- 医疗健康诊断工具。
- 营养治疗或疾病管理服务。
- 外卖、食品销售或到店餐饮服务。

## 4. 审核备注

可直接粘贴以下内容：

```text
Humi 是一款家庭晚饭安排工具，核心功能无需注册即可体验。

建议审核路径：
1. 打开小程序，进入“今晚吃什么？”首页；
2. 点击“帮我安排晚饭”查看推荐；
3. 点击“换一组”并选择反馈原因；
4. 点击“就吃这组”进入今晚菜单；
5. 点击“看看还要买什么”查看食材清单；
6. 可继续查看“一周计划”和“自己挑”。

推荐服务不可用时，页面会自动使用本地规则给出另一组菜单，不影响核心体验。
本产品仅提供家庭菜单、采购清单和库存整理，不提供医疗诊断或专业营养建议。
```

## 5. 隐私保护指引填写对照

后台选项名称可能随平台调整，应以实际页面为准。按最接近的选项如实选择。

| 产品数据 | 是否收集 | 使用目的 | 触发方式 |
| --- | --- | --- | --- |
| 邮箱账号 | 登录时 | 注册、登录、找回会话、家庭协作 | 用户主动填写 |
| 密码 | 登录时 | 账号认证，由认证服务处理 | 用户主动填写 |
| 家庭成员邮箱 | 邀请时 | 添加家庭成员 | 用户主动填写 |
| 家庭人数、是否有儿童 | 可选 | 调整份量和菜单建议 | 用户主动填写 |
| 口味、忌口、过敏、饮食目标 | 可选 | 避开不合适的菜并改善推荐 | 用户主动填写 |
| 今晚菜单和一周计划 | 使用功能时 | 保存和同步饮食安排 | 用户主动操作 |
| 食材清单和勾选状态 | 使用功能时 | 生成并保存采购清单 | 用户主动操作 |
| 厨房库存、数量、到期日 | 可选 | 库存管理和优先消耗提醒 | 用户主动填写 |
| 推荐反馈 | 换一组时 | 调整后续推荐 | 用户主动选择 |
| 使用事件、随机会话标识 | 使用功能时 | 统计核心流程、发现故障 | 自动生成，不读取微信身份 |

当前不收集：

- 微信头像、昵称和 OpenID。
- 手机号。
- 精确位置。
- 通讯录。
- 相册内容。
- 摄像头和麦克风。
- 支付信息。

若后续接入微信登录、手机号绑定、相册保存权限或订阅消息，必须先更新隐私保护指引和隐私政策。

## 6. 数据处理说明

### 游客

- 菜单、计划、清单、库存和家庭画像主要保存在当前 WebView 的本地存储。
- 仍可能发送不含微信身份的产品使用事件。
- 用户清理微信缓存或 WebView 数据后，本地数据可能丢失。

### 登录用户

- 邮箱账号和家庭数据可同步到云端。
- 数据按家庭空间隔离。
- 推荐请求仅发送完成推荐所需的菜谱候选、家庭偏好、库存、历史安排和反馈。
- 家庭成员邮箱已从推荐上下文中移除，推荐服务只接收匿名成员编号。
