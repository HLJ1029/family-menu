# Humi 微信小程序上线计划 V1

更新日期：2026-07-03

本文档记录 Humi 小程序 WebView 首发的当前执行口径。更细的验收闸门、风险和材料清单见 `docs/miniprogram-launch-readiness.md` 与 `docs/miniprogram-review-materials.md`。

## 项目现状

当前产品已完成：【今晚】推荐、自己挑、想连排几天、食材清单、家庭画像、后台已有隐形维护、家庭感觉征集、买菜认领、微信登录、Humi API 同步、DeepSeek 精准推荐额度闸门、H5 与小程序 WebView 部署。

目标：

验证用户是否愿意持续使用 Humi 解决“今晚吃什么”，以及是否愿意把家人拉进同一个吃饭协作空间。

## 品牌统一

- 品牌名：Humi
- Slogan：让每顿饭都有安排
- 视觉口径：黑、白、灰；不新增彩色含义。
- 产品语气：温暖、家庭感、烟火气、轻陪伴。

## MVP 上线策略

采用 WebView 承载现有 H5，不在首发阶段重写原生页面。

MVP 包含：

- 【今晚】推荐、换一组、不想吃、今晚就做。
- 【自己挑】小红书式菜品浏览，一键补进今晚。
- 想连排几天，作为【今晚】辅助入口。
- 【清单】自动汇总三餐食材，支持勾选、分享和买菜认领。
- 【我的家】家庭成员、饭线索、想吃池子、忌口和账号同步。
- 感觉征集、家庭邀请和买菜认领小程序卡片。
- 微信登录与游客体验。
- 基础推荐免费无限；精准推荐走服务端额度闸门和缓存复用。

MVP 不包含：

- 微信支付、会员、商城。
- 社区、UGC 和公开内容广场。
- 原生页面重构。
- 订阅消息。
- 独立后台已有维护页或软口味主动维护表。

## AI 推荐策略

基础推荐对所有用户免费开放，不按次数收费。精准 API 推荐作为更高质量体验，使用服务端额度闸门、Plus 权益和缓存复用控制成本。

推荐架构：

1. 本地规则推荐。
2. 登录家庭有额度或 Plus 时调用 DeepSeek 精准推荐。
3. 异常、无额度或接口失败时自动回退本地推荐。

核心指标：

- 推荐次数。
- 换一组次数。
- 今晚就做次数。
- 问问大家发起次数。
- 家人投感觉次数。
- 买菜认领次数。

重点关注推荐接受率、协作完成率和 7 日留存。

## 登录策略

- 游客模式优先，先体验核心推荐链路。
- 普通启动小程序先进入【今晚】，不先进入登录页。
- 微信登录用于小程序主身份，已由 Humi API 承接。
- 登录后置：保存、同步、加入家庭、发起邀请或跨设备恢复时再引导登录。
- 邮箱登录不作为小程序普通用户入口，仅保留为开发测试和海外预留能力。
- 手机号绑定作为增强身份，用于换设备找回、家庭协作安全和后续支付/会员。

## 微信小程序审核

服务类目必须以微信公众平台当前可选项和主体资质为准，优先选择生活工具、效率管理或家庭生活相关类目；不要把 Humi 描述成医疗、营养治疗、外卖或食品销售服务。

避免出现：

- DeepSeek
- Supabase
- PWA
- AI Agent
- 大模型
- 规则引擎

统一使用：

- 智能推荐
- 晚饭安排
- 家庭协作
- 买菜清单
- 家庭饮食助手

## 产品成功标准

不是用户是否立刻付费，而是用户是否连续打开，并愿意把家庭协作动作跑起来。

MVP 成功标准：

- 10-20 个家庭完成首批灰度反馈。
- 推荐接受率 > 50%。
- 至少一部分家庭完成“问问大家”或买菜认领分享。
- 7 日留存 > 25%。
- 30 日留存 > 10%。
