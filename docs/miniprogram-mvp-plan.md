# 呼米小程序 MVP 准备方案

## 当前策略

先用小程序 `web-view` 承载现有呼米 H5/PWA，快速验证“今晚吃什么”这个核心价值。暂不全量重写小程序，也不先接微信登录。

更细的上线材料、域名、后台配置和审核清单见 `docs/miniprogram-launch-readiness.md`。

## 为什么先 web-view

- 当前 React 版本已经打通推荐、菜单、计划、清单和库存。
- V3 品牌升级后，PWA、小程序、后续 iOS 可以共用同一套产品语言。
- web-view 壳最快，适合在微信生态里做早期 MVP 验证。
- 真正确定用户愿意使用后，再重写小程序核心页。

## 需要准备

- 正式 HTTPS 域名，不建议长期依赖 GitHub Pages 域名。
- 小程序后台配置业务域名。
- 域名校验文件可放到站点根目录。
- 隐私政策和用户协议：
  - `/family-menu/privacy.html`
  - `/family-menu/terms.html`
- 小程序首页先进入呼米首页，不先进入登录页。
- 页面内不要出现旧品牌名、模型名、后端服务名、`PWA`、`规则引擎` 等技术词。

## 小程序第一版范围

- 小程序壳打开呼米首页。
- 用户能看到“今晚吃什么？”和今晚推荐。
- 用户能查看“自己挑”、一周计划、食材清单。
- 保存、登录、家庭协作暂时沿用 H5 内现有能力。
- 微信登录暂不接入，只保留准备项。

## 第二阶段再考虑

- 原生小程序重写首页。
- 原生小程序重写菜单详情和食材清单。
- 接入微信登录。
- 接入分享卡片、订阅消息或家庭邀请。

## 品牌约束

- 小程序、PWA、后续 iOS 统一叫「呼米」。
- 核心问题统一是「今晚吃什么？」。
- 产品语气保持温暖、家庭感、烟火气、轻陪伴。
- 不对用户展示模型、供应商、接口、规则、fallback 等实现细节。
