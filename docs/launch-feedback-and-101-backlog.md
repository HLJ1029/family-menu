## 1. 首批用户邀请文案

### 简短版

```text
我做了一个家庭吃饭安排小程序 Humi，想请你帮忙试一下。

它主要解决“今晚吃什么”：打开后可以先拿到晚饭推荐，也能问问家人想吃什么，自动生成买菜清单。

你试 5 分钟就行，重点帮我看：
1. 推荐的菜想不想吃
2. 买菜清单有没有用
3. 问家人/分享清单这一步有没有顺
4. 哪一步让你困惑或卡住

有问题直接截图发我，越真实越好。
```

### 稍正式版

```text
Humi 1.1 正在做首批小范围测试，想邀请你体验一下。

使用路径：
打开小程序 → 看【今晚】推荐 → 今晚就做/换一组 → 问问大家想吃啥 → 看【清单】 → 分享或认领买菜。

它不是专业营养或医疗建议，只是帮家庭把今晚吃什么、买什么、谁来买整理清楚。

如果你愿意反馈，麻烦告诉我：
- 哪个推荐你会真的做
- 哪个推荐不符合你家口味
- 哪一步最麻烦
- 问问大家/清单分享有没有打不开、回传失败或登录卡住
```

## 4. 问题分级标准

- P0：打不开、无法登录且无法游客继续、推荐/菜单/清单主链路断、审核合规问题。
- P1：能用但明显影响继续使用，比如同步不清楚、关键按钮看不懂、推荐明显不符合家庭场景。
- P2：视觉、文案、性能、结构优化，不影响完成核心任务。

## 5. 首批验证目标

- 首批用户：10-20 个家庭或个人。
- 每人至少完成一次“【今晚】推荐 → 今晚菜单 → 清单”，最好再完成一次“问问大家”或买菜认领分享。
- 重点看 3 个指标：
  - 推荐是否愿意接受。
  - 清单是否真的减少思考。
  - 用户是否愿意第二天再打开。
