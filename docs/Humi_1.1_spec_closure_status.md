## 已落地

### 结构与三餐

- 三个主 tab 固定为“今晚 / 清单 / 我的家”；菜品库、今晚菜单、周计划、日历和营养分析均为子页。
- “全部菜品库”展示完整菜品流，已安排菜置顶；可从今晚、推荐页和我的家进入。
- 早餐进入完整菜品库轻量选择；午餐先选来源，在家做时再进入完整菜品库；不再默认写入紫菜蛋花汤。
- 三餐食材合并进入清单；清单勾选和晚饭确认继续驱动后台食材状态，界面不暴露库存维护页。
- 库存为空时提供可跳过的“告诉我家里有啥”单行入口；填写后只作为推荐加分项，不变成库存维护页或使用门槛。
- 早餐、午餐来源和晚饭确认已共同进入家庭动态与三餐画像；记录会保留操作者并在【我的家】合并成当天吃饭回执。
- 周计划和日历的空白天已降噪；空日不显示完成态、0 道徽标或空营养环。
- 清单、今晚菜单和我的家已移除常驻自我介绍、重复区块和卡片套卡片；主 tab 不再显示返回按钮。
- 登录后的家庭确认只询问人数、孩子和硬忌口；规划模式、买菜接受度和软口味均不再要求用户维护。

### 感觉征集闭环

- 主厨可选择征集对象、兜底感觉并查看征集单模板。
- 家人从小程序卡片进入后可免登录点感觉、补一句并回传。
- 感觉、买菜、菜单和想吃池四种有效分享页已按 390×844 手机视口验收；核心选择、菜单内容和主操作均完整位于首屏，并有自动化几何断言防止回退。
- 四种分享落地页在链接失败时提供“重试 / 回到 Humi”双退路；加载和错误状态不再用大插画占满首屏。
- 等待、刷新、截止时间自动收口、无人回复兜底、我自己做主、都不想吃返回感觉选择均已实现。
- 菜单展示揉合理由；勾选后一次写入今晚菜单和买菜清单。
- 晚间轻确认支持做了、换了别的、出去吃了和不记录；旧的食材扣减标记会正确清理或重建。
- 感觉、想吃、买菜和晚饭确认会沉淀为家庭动态与画像线索；零线索时不铺满空画像卡。

### 家庭协作

- 买菜清单可生成小程序分享页；家人可免登录认领、拒绝和勾选，主厨刷新后同步结果。
- 想吃池可生成小程序分享页；家人可免登录写菜，主厨刷新后决定是否安排。
- 家庭邀请可生成原生小程序卡片；家人点开进入指定家庭，登录后成为正式成员，不再落回普通首页。
- 临时参与者登录后会自动合并为正式成员；失败时仍保留手动重试，历史感觉、认领和想吃记录保留合并时间。
- Household 状态已按家庭保存；主厨与正式家人读取同一份菜单、清单、忌口和画像。
- 一个用户可创建或加入多个家庭，并在【我的家】切换；家庭之间的状态隔离由 API 冒烟覆盖。
- 主厨/家人权限已在后端和界面同时限制：家人不能生成家庭邀请或发起主厨专属协作请求。
- 分享请求和参与记录保留 `householdId`、`ownerId`、临时身份和合并时间等归属字段。
- 我的家已收成协作概览、真实动态/画像、成员、家庭邀请/切换、账号和设置；完整菜品库入口保留。

### 推荐质量与成本

- 基础推荐保持免费本地路径；硬忌口同时覆盖过敏和明确不吃项。
- 精准推荐要求 Humi 身份，支持尝鲜额度、缓存复用和额度不足响应；匿名请求不会消耗精准 API。
- 免费协作、感觉征集、清单和基础推荐没有次数墙。

### 小程序分享工程

- H5 分享动作通过 WebView 消息桥打开原生分享子页，导航失败时有重定向兜底。
- 原生分享页首屏使用 `open-type="share"` 按钮，并为感觉、清单、菜单、想吃池和家庭邀请生成对应卡片路径。
- 微信开发者工具已使用正式 AppID 编译，当前显示 0 个工程问题。
- 已删除开发者工具判定无效的 page.json 分享字段；WebView 首页不再声明不支持的朋友圈分享。

### 生产级保护与视觉门禁

- 免登录协作写接口增加宽松的 IP 限流；异常高频写入返回 429，正常家庭协作不按次数收费。
- API JSON 请求体限制为 64KB；超限请求排空连接后返回 413，不影响后续连接复用。
- Web、小程序壳与分享海报的 UI 色值已归一为黑白灰；删除暖色/荧光径向装饰，并增加 `validate:palette` 静态门禁。
- 日历“添加菜品”、清单餐次数量、菜谱步骤与统计图标等黑底元素已补足前景对比度，并由浏览器回归覆盖核心操作。

## 仍需产品与平台方案

### 订阅与正式付费

当前已完成基础推荐 / 精准推荐的技术分层、尝鲜和成本控制，但没有接正式订阅支付。支付、权益持久化和恢复购买属于高风险范围，需要单独确认产品方案与平台方案。

- 在真机微信中验证五种小程序卡片的发出、落地和回传（家庭邀请需登录加入，其余参与页先免登录）。
- 用户验收通过后，再执行版本上传、审核材料和 1.1 发布。
