# Humi 微信登录无感切换热修设计

## 目标

修复 H5 已更新、微信小程序原生壳仍为旧版本时，点击“完善身份”永久停在加载态的问题；同时保留新壳中的微信原生头像与昵称流程。

## 交互设计

- 未登录用户在小程序内点击“微信登录”时，优先跳转 `/pages/identity/index?action=login`。
- 新壳中由 `wx.login` 静默换取 Humi 会话，随后显示微信原生头像选择与昵称输入控件。
- 原生跳转失败时，H5 不得永久加载：尝试旧消息桥，并在短超时后恢复按钮，提示退出重进或更新小程序。
- 已经持有“不完整”Humi 会话的用户不再重复登录。H5 直接显示身份兜底页，可填写昵称并选择 Humi 头像后进入应用。
- 身份兜底页保留“使用微信头像和昵称”入口；新壳可进入原生页，旧壳失败时仍可留在当前页完成身份。

## 数据与兼容性

- `PUT /identity/profile` 接收 `displayName` 与可选 `avatarKey`。
- 服务端继续由现有 Humi Store 保存，不引入 Supabase。
- 旧会话、旧壳和新壳均复用现有 session 格式；不增加敏感参数到 URL。

## 验收

- 旧的不完整会话可在 H5 保存昵称与头像，刷新后为完整身份。
- 新壳点击微信登录进入原生身份页。
- 原生跳转失败时，按钮在超时后可再次点击，不出现永久转圈。
- API smoke、H5 微信 WebView 回归、构建、线上检查全部通过。
